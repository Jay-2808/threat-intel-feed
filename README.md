# Threat-intel-feeds

A collection of **sample threat intelligence feeds** provided in multiple formats for developers and security researchers.

These datasets can be used to test:

- Threat intelligence ingestion pipelines
- SIEM integrations
- Log parsers
- Data processing scripts
- Security analytics tools

The data is derived from publicly available threat intelligence datasets and converted into developer-friendly formats.

---

## Available Formats

This repository provides threat indicators in different formats:

| Format | Description |
|------|-------------|
| CSV | Structured datasets for database import and analytics |
| JSON | Structured feed format for APIs and applications |
| TXT | Simple flat feeds for quick testing and parsing |

---

## Example Use Cases

Developers can use this repository to:

- Test threat feed ingestion systems
- Benchmark data pipelines
- Build security dashboards
- Develop detection rules
- Simulate real-world cyber threat feeds

---

## Example Indicator Types

Datasets may contain indicators such as:

- IP addresses
- Domains
- URLs
- File hashes (MD5, SHA256)
- Malware campaign names
- Threat activity labels

---

## Sample Data Sources

Some datasets are derived from threat intelligence formats such as **STIX** and converted into simplified formats for testing.

---

## Disclaimer

These datasets are provided **only for development and testing purposes**.

They should **not be used directly for production threat blocking** without proper validation and enrichment.
